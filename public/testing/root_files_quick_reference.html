How to Use

Method 1: File Browser (Recommended)

Navigate to the root proxy URL and browse available files visually:

https://yourdomain.com/testing/root_proxy.php

Method 2: Direct File Access

Access a specific test file directly by adding the file parameter:

https://yourdomain.com/testing/root_proxy.php?file=test_image_variations.php

Method 3: Via Testing Dashboard

Access through the main testing dashboard:

https://yourdomain.com/testing/

Click on "Root Files Browser" in the "Root Test Files" section

Adding New Files to Whitelist

To allow access to a new test file:

  1. Edit public/testing/root_proxy.php
  2. Add the filename to the $allowedFiles array
  3. Upload the updated file to production
Security Warning: Never add sensitive files (config.php, .env, database credentials, etc.) to the whitelist!
